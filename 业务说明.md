# 🚀 综合设计项目：基于 Flutter + SpringBoot 的即时通信 (IM) 软件开发

## 1. 项目概述
本项目是一个极具未来感与人文温度的现代即时通信 App。前后端分离开发，前端使用 Flutter 构建跨平台移动应用，后端使用 Spring Boot 提供 RESTful API 并通过 WebSocket 维持长连接实现实时通信。

### 1.1 核心业务需求 (Business Requirements)
*   **鉴权中心：** 用户注册、登录校验（基于 JWT），设备状态保持。
*   **实时通信 (核心)：** 支持私聊（1v1），基于 WebSocket 实现消息的实时收发。
*   **离线与历史记录：** 消息持久化到数据库，支持离线消息拉取和历史聊天记录漫游。
*   **好友管理：** 搜索用户、添加好友、好友列表展示。
*   **状态与提醒：** 实时在线状态展示、未读消息红点（非正圆液态形状）。
*   **UI/UX 规范：** 暖色渐变背景、Glassmorphism (毛玻璃/玻璃态)、不对称有机形状 (Organic Shapes)、微动态交互。

---

## 2. 技术栈与框架选型 (Tech Stack)

### 2.1 移动端前端 (Flutter)
*   **UI 框架：** Flutter (Dart)
*   **状态管理：** Provider 或 GetX (推荐使用 GetX 以快速实现响应式状态管理)
*   **网络请求：** `dio` (处理 RESTful API)
*   **长连接：** `web_socket_channel` (处理实时消息)
*   **本地存储：** `shared_preferences` (存储 JWT Token 和简单配置)

### 2.2 服务端后端 (Spring Boot)
*   **核心框架：** Spring Boot 3.x (Java)
*   **持久层框架：** MyBatis-Plus (快速实现 CRUD)
*   **长连接：** `spring-boot-starter-websocket`
*   **安全与鉴权：** JWT (JSON Web Token)
*   **数据库：** MySQL 8.0 (数据持久化)
*   **缓存与中间件：** Redis (在线状态管理、未读消息计数、性能缓冲)
*   **JSON 解析：** Jackson / FastJSON2

---

## 3. 数据库设计 (Database Schema)

### 3.1 MySQL 核心表结构
业务数据持久层，采用 `utf8mb4` 字符集以支持 Emoji。
```sql
-- 1. 用户信息表
CREATE TABLE `user_info` (
  `user_id` BIGINT AUTO_INCREMENT PRIMARY KEY,
  `username` VARCHAR(50) NOT NULL UNIQUE,
  `password` VARCHAR(255) NOT NULL,
  `avatar_url` VARCHAR(255) DEFAULT NULL,
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 2. 好友关系表
CREATE TABLE `friend_relation` (
  `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
  `user_id` BIGINT NOT NULL,
  `friend_id` BIGINT NOT NULL,
  `status` TINYINT DEFAULT 0 COMMENT '0-待通过, 1-已通过',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_user_friend` (`user_id`, `friend_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 3. 聊天记录表 (核心)
CREATE TABLE `chat_message` (
  `msg_id` BIGINT AUTO_INCREMENT PRIMARY KEY,
  `sender_id` BIGINT NOT NULL,
  `receiver_id` BIGINT NOT NULL,
  `chat_type` TINYINT DEFAULT 1 COMMENT '1-私聊',
  `msg_type` TINYINT DEFAULT 1 COMMENT '1-文本, 2-图片',
  `content` TEXT NOT NULL,
  `is_read` TINYINT DEFAULT 0 COMMENT '0-未读, 1-已读',
  `send_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX `idx_sender_receiver` (`sender_id`, `receiver_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.2 Redis 缓存设计 (性能优化)

- **在线状态管理：** `Hash` 结构。Key: `im:user:status`, Field: `user_id`, Value: `1` (在线) / `0` (离线)。
- **未读消息计数：** `Hash` 结构。Key: `im:unread:{receiver_id}`, Field: `sender_id`, Value: `未读数量`。

## 4. WebSocket 通信协议规范 (非常重要)

前后端通过 WebSocket 传输数据时，统一使用 JSON 格式。

### 4.1 客户端发送消息格式 (Client -> Server)

JSON

```
{
  "type": "CHAT",          // 消息类型: CHAT(聊天), HEARTBEAT(心跳)
  "senderId": 1001, 
  "receiverId": 1002,
  "msgType": 1,            // 1-文本, 2-图片
  "content": "Hello, 晚上一起打羽毛球吗？"
}
```

### 4.2 服务端推送消息格式 (Server -> Client)

JSON

```
{
  "msgId": 9527,
  "type": "CHAT",
  "senderId": 1001,
  "receiverId": 1002,
  "msgType": 1,
  "content": "Hello, 晚上一起打羽毛球吗？",
  "sendTime": "2026-05-01T20:15:00"
}
```

## 5. 开发阶段执行计划 (To Cursor)

Cursor，请理解上述上下文，我们将按以下步骤进行开发协作：

- **Phase 1 (Backend CRUD):** 编写 Spring Boot 后端的 User 和 Friend 模块的 RESTful API，并打通 MySQL 读写。
- **Phase 2 (WebSocket & Redis):** 在后端建立 `WebSocketHandler`，集成 JWT 鉴权，实现 Redis 用户状态上下线管理及消息转发逻辑。
- **Phase 3 (Flutter State):** 在现有的 Flutter UI 基础上，引入 GetX/Provider 构建 `AuthService` 和 `ChatController`，将硬编码数据替换为响应式变量。
- **Phase 4 (Integration):** 编写 Flutter `WebSocketService`，联调本地 Spring Boot 服务，跑通真实的消息收发与 UI 更新。
