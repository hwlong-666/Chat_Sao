## 🔴 P0 — 核心缺失功能（IM 基础体验）

### 1. 图片消息
**现状**: `msgType` 只有 `0=文本` 和 `3=语音`，无法发送图片
**目标**: 支持发送/接收图片消息，聊天中渲染图片缩略图，点击可全屏预览

| 端 | 任务 |
|----|------|
| 后端 | `FileController.upload` 已支持通用文件上传，无需改动 |
| 后端 | `ChatWebSocketHandler` 已支持 `msgType` 转发，无需改动 |
| 前端 | `ChatDetailScreen` 底部输入栏增加图片选择按钮（`image_picker`） |
| 前端 | 选中图片后调用 `FileUploadService.uploadImage()` 上传，获取 URL |
| 前端 | 通过 WebSocket 发送 `{ receiverId, content: url, msgType: 2 }` |
| 前端 | 消息列表中 `msgType==2` 渲染图片缩略图（`CachedNetworkImage`） |
| 前端 | 点击缩略图进入全屏预览页面 |
| 前端 | `ChatListScreen` 会话预览显示 `📷 图片消息` |
| 前端 | `pubspec.yaml` 新增 `image_picker`、`cached_network_image` 依赖 |

### 2. 用户个人资料编辑 & 头像上传
**现状**: `UserInfo` 有 `avatarUrl` 字段但始终为 `null`；设置页头像/昵称是硬编码的 `User.currentUser`（Mock 数据）；无法修改个人信息
**目标**: 用户可上传头像、修改昵称，资料实时同步

| 端 | 任务 |
|----|------|
| 后端 | 新增 `PUT /api/user/profile` 接口（接收 `username`、`avatarUrl`） |
| 后端 | 新增 `POST /api/user/avatar` 接口（上传头像文件，返回 URL） |
| 后端 | `UserInfoService` 增加更新资料方法 |
| 前端 | 设置页头像旁的编辑按钮改为可点击，弹出编辑资料弹窗 |
| 前端 | 编辑弹窗支持：修改昵称、上传头像（裁剪后上传） |
| 前端 | `AuthService` 登录时保存并缓存 `avatarUrl`，编辑后同步更新 |
| 前端 | 聊天页/联系人页/会话列表页 均使用真实头像 URL 渲染 |
| 前端 | `User.dart` 移除全部 Mock 用户常量，改为动态数据 |

### 3. 删除好友
**现状**: 只能添加/接受/拒绝好友，无法删除好友
**目标**: 好友详情页或联系人页支持删除好友，双方关系同时解除

| 端 | 任务 |
|----|------|
| 后端 | 新增 `DELETE /api/friend/remove` 接口（双向删除 `friend_relation` 记录） |
| 后端 | `FriendRelationService` 增加 `removeFriend()` 方法 |
| 前端 | 联系人页好友卡片增加删除按钮（左滑删除或长按菜单） |
| 前端 | 删除后刷新好友列表和会话列表 |

## 🟡 P1 — 重要增强功能（提升可用性）

### 4. 离线消息同步
**现状**: 用户离线期间收到的消息，重新上线后只能通过进入聊天页拉取历史才知道，会话列表不会自动更新
**目标**: 用户重连 WebSocket 后，自动拉取离线期间的所有新消息，更新会话列表和未读数

| 端 | 任务 |
|----|------|
| 后端 | 新增 `GET /api/chat/offline` 接口（返回所有未读消息摘要，按发送者分组） |
| 后端 | WebSocket 重连后由前端主动调用该接口 |
| 前端 | `WebSocketService` 重连成功后，自动调用离线消息接口 |
| 前端 | 收到离线消息后更新会话列表（最后消息、未读数） |
| 前端 | 如果当前正在某聊天页，自动追加离线消息到列表 |

### 5. 消息发送状态指示
**现状**: 消息发出后没有"发送中/已发送/已读"状态标识
**目标**: 每条消息右侧显示 ✓（已发送）/ ✓✓（对方已读）

| 端 | 任务 |
|----|------|
| 后端 | WebSocket 消息推送中增加 `isRead` 字段（已有） |
| 后端 | 新增 WebSocket 推送类型 `{ type: "READ", friendId }`（对方已读通知） |
| 前端 | `ChatMessageItem` 增加 `sendStatus` 字段（sending/sent/read） |
| 前端 | 发送消息时先显示 sending 状态，WebSocket 回传后改为 sent |
| 前端 | 收到 READ 通知后，将该好友所有已发消息标记为 read |
| 前端 | 消息气泡右下角渲染 ✓ / ✓✓ 图标 |

### 6. 消息搜索
**现状**: 无法搜索聊天记录
**目标**: 在聊天页顶部增加搜索入口，可按关键词搜索历史消息

| 端 | 任务 |
|----|------|
| 后端 | 新增 `GET /api/chat/search?keyword=&friendId=&limit=&offset=` 接口 |
| 后端 | SQL 使用 `LIKE '%keyword%'` 模糊匹配 `content` 字段 |
| 前端 | 聊天详情页头部增加搜索图标 |
| 前端 | 点击后弹出搜索框，输入关键词实时搜索 |
| 前端 | 搜索结果列表点击可跳转到该消息在聊天记录中的位置 |

### 7. 好友在线状态展示
**现状**: 后端 Redis 有在线状态数据，但前端只在聊天详情页显示 WebSocket 连接状态（自己的），无法看到好友是否在线
**目标**: 联系人页和会话列表页显示好友在线/离线状态

| 端 | 任务 |
|----|------|
| 后端 | 新增 `GET /api/friend/online-status` 接口（返回好友在线状态列表） |
| 后端 | 或在 `/api/friend/list` 和 `/api/chat/sessions` 返回中增加 `isOnline` 字段 |
| 前端 | 联系人页好友头像右下角显示绿点（在线）/ 灰点（离线） |
| 前端 | 会话列表页好友头像同样显示在线状态 |
| 前端 | 聊天详情页头部显示好友在线状态（而非自己的连接状态） |

### 14. 安全加固
**现状**: CORS 全开放、无接口限流、密码明文传输
**目标**: 基本安全防护

| 端 | 任务 |
|----|------|
| 后端 | CORS 收紧为前端域名白名单 |
| 后端 | 接口限流（`@RateLimiter` 或 Guava RateLimiter） |
| 后端 | 登录接口增加验证码或防暴力破解机制 |
| 后端 | JWT Token 刷新机制（当前 Token 过期后需重新登录） |
| 后端 | 敏感配置（数据库密码、JWT Secret）移至环境变量 |
| 前端 | HTTPS 适配（生产环境） |

### 15. 日志与监控
**现状**: 后端有基本 log，无结构化日志和监控

| 端 | 任务 |
|----|------|
| 后端 | 引入 `Logback` 结构化日志（JSON 格式） |
| 后端 | 关键操作（登录/注册/消息发送）记录审计日志 |
| 后端 | 接入 `Spring Boot Actuator` 健康检查端点 |
